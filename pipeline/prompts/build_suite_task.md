# Task: Build Robot Framework Test Suite from BDD IR

## Objective
Consume a BDD Intermediate Representation and generate complete, lint-clean Robot Framework test suite and keyword resource files.

## Input Parameters

### BDD IR File
**Path**: `{IR_FILE}`

This is the BDD IR generated by `bdd-ir-builder`, conforming to `pipeline/schemas/bdd-ir.schema.json`.

## Execution Steps

1. **Pre-flight Validation**
   - Confirm IR file path is `{IR_FILE}`
   - Confirm file exists
   - Read and parse the IR JSON
   - Validate it contains `feature.scenarios` with resolved steps
   - Extract feature name (convert to `snake_case` for file names)

2. **Generate Test Suite File**
   - Create `robot/suites/{FEATURE_NAME}_suite.robot`
   - Include `*** Settings ***` with:
     - All library imports from IR
     - All resource imports from IR
     - Metadata tags
   - Include `*** Variables ***` section with placeholder variables
   - Include `*** Test Cases ***` with:
     - One test case per scenario
     - Scenario Outline rows expanded into separate test cases
     - Proper BDD keywords (Given, When, Then, And, But)
     - Tags from Gherkin, plus `generated` tag
   - Include `*** Keywords ***` section if Background steps exist

3. **Generate Resource File (if needed)**
   - If `unresolved_count > 0`:
     - Create `robot/resources/keywords/generated/{FEATURE_NAME}_keywords.resource`
     - Include stub implementations for all NEEDS_GENERATION keywords
     - Each stub has:
       - Documentation from generation_hint
       - Argument list
       - Log statement
       - Fail statement with NOT IMPLEMENTED message

4. **Validate & Write**
   - Ensure all imports are listed
   - Ensure all test cases have Tags as first line
   - Ensure no hardcoded environment values
   - Write final files to `robot/suites/` and `robot/resources/keywords/generated/`

## Expected Outputs

- **Suite file**: `robot/suites/{FEATURE_NAME}_suite.robot`
- **Resource file**: `robot/resources/keywords/generated/{FEATURE_NAME}_keywords.resource` (if unresolved keywords exist)
- **Robocop report**: `pipeline/robocop_report.json`

### Example Suite File Structure
```robot
*** Settings ***
Library    SeleniumLibrary
Resource   robot/resources/keywords/common.resource
Metadata    generated_by    bdd-to-robot-pipeline
Metadata    source_feature    bdd/features/login.feature

*** Variables ***
${LOGIN_URL}    # TODO: set value
${ADMIN_USER}    # TODO: set value

*** Test Cases ***
Successful Login With Valid Credentials
    [Tags]    smoke    auth    generated
    Given    Open Login Page
    When     Enter Credentials    admin    password123
    Then     Verify Dashboard Is Displayed

*** Keywords ***
Run Background Steps
    Start Application
```

## Key Rules

- **Expand Scenario Outlines**: Each row becomes a separate test case
- **Stub unresolved keywords**: Every NEEDS_GENERATION step must have a stub
- **Self-correct on lint failures**: Fix and re-lint up to 3 times
- **No semantic changes**: Only fix formatting/structure issues, not logic
- **All stubs fail loudly**: Ensure `Fail` statement in every stub for clarity

## Clean Success Criteria

✅ Suite file written to `robot/suites/{FEATURE_NAME}_suite.robot`
✅ Resource file written (if needed) to `robot/resources/keywords/generated/{FEATURE_NAME}_keywords.resource`
✅ Robocop lint is clean (zero errors, zero warnings)
✅ All stubs included for NEEDS_GENERATION steps
✅ All imports from IR are included
✅ Do not delegate further; return to orchestrator

## Failure Handling

If Robocop issues cannot be resolved after 3 attempts:
- Do not delegate
- Write `pipeline/robocop_report.json` with all remaining issues
- Return the report to the orchestrator for human review
- List which issues need manual intervention
